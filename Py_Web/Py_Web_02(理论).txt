软件开发生命周期
        不需要测试人员参与的：
                需求调研
        测试人员全程都要参与：
                需求分析评审
                软件架构设计
                编码
                单元测试
                集成测试
                系统测试
                预发布系统测试
                上线

        遇到的困难：
                接口文档不完善；需求变更后接口文档没有及时更新
        怎么解决：
                每次需求评审后都会和开发讨论更改的接口文档协议

测试人员分类：
        黑盒测试（功能测试）：功能符合用户的需求。

        白盒测试：知道系统的代码逻辑，验证符合用户的需求。

        冒烟测试：主功能，主路径测试

        性能测试：模拟多种峰值和负载去测试各项性能指标，常用工具jmeter/loaderunner

        自动化测试：人工测试转为工具自动执行

自动化测试和手工测试优缺点
        1.自动化测试方便，自动验证功能是否有问题
        2.更号的利用资源，节约人力成本
        3.覆盖更全，增加软件的稳定性
        场景：需求变更不频繁，项目周期长，稳定性强，环境要隔离独立（开发一套环境，测试一套环境）
              如果整个系统需求变更多，只能抽取部分功能做自动化测试

        语言选择：
                python   java   ruby   php  （打造唯一性，不容易被取代）
